# معماری عکس برای نسخه عملیاتی Modline

این نسخه دیگر از لینک خارجی برای Hero و محصول استفاده نمی‌کند. عکس‌های نمونه به صورت WebP داخل پروژه ذخیره شده‌اند تا روی سرور ایران سریع‌تر لود شوند.

## مسیرهای اصلی

```text
/public/uploads/banners/       عکس‌های Hero و بنرها
/public/images/fashion/        عکس‌های seed محصولات و دسته‌بندی‌ها
/public/uploads/products/      عکس‌هایی که از پنل ادمین آپلود می‌شوند
/public/images/IMAGE-CREDITS.json  منبع عکس‌های نمونه
```

## فلو آپلود محصول

```text
Admin Login
↓
Products
↓
Add Product یا Edit
↓
انتخاب عکس JPG/PNG/WebP زیر ۶MB
↓
POST /api/admin/uploads
↓
ذخیره در /public/uploads/products
↓
برگشت URL عکس
↓
ذخیره URL در محصول
↓
نمایش در سایت با local path
```

## توصیه سرعت

- Hero دسکتاپ: 1600×900 و حدود 120 تا 250KB
- Hero موبایل: در آینده جدا اضافه شود؛ فعلاً object-cover استفاده شده است
- عکس محصول: نسبت 3:4 و حدود 80 تا 250KB
- در production عکس را از URL خارجی نخوان؛ عکس را local یا روی Object Storage داخلی/ایرانی ذخیره کن
- عکس در دیتابیس ذخیره نشود؛ فقط مسیر عکس در دیتابیس/JSON ذخیره شود

## نکته محتوایی ایران

برای کاهش ریسک عملیاتی، Hero و دسته‌بندی‌ها بیشتر محصول‌محور، بوتیک‌محور، پوشیده و بدون نمایش نامناسب انتخاب شده‌اند. برای فروش واقعی بهتر است عکس اختصاصی محصولات خودت جایگزین نمونه‌های استوک شود.
