# مدلاین — نسخه عملیاتی ایران‌محور

این نسخه برای فارسی/RTL، سرعت بهتر داخل ایران، و حذف بن‌بست‌های اصلی کاربر و ادمین آماده شده است.

## تصمیم مهم برای عکس‌ها
برای سرعت بالا، عکس‌ها نباید از لینک‌های خارجی مثل Unsplash/Google/CDN خارجی لود شوند. عکس‌های سایت در این نسخه به فایل‌های local WebP داخل مسیر زیر منتقل شده‌اند:

```text
/public/images/fashion/
```

برای نسخه واقعی، عکس‌های محصول و بنر را با عکس‌های واقعی و فشرده‌شده WebP در همین مسیر جایگزین کنید. بهترین حالت برای ایران: فایل local روی سرور ایران یا CDN داخلی ایران.

## User Flow واقعی

```text
ورود به سایت فارسی
↓
دیدن بنر، دسته‌بندی، محصولات
↓
جستجو / فیلتر / صفحه محصول
↓
انتخاب سایز، رنگ، تعداد
↓
افزودن به سبد خرید
↓
ورود یا ثبت‌نام
↓
ثبت آدرس ارسال
↓
ثبت سفارش روی سرور
↓
کسر موجودی محصول
↓
صدور رسید سفارش با وضعیت pending
↓
پرداخت/تأیید مالی و تغییر وضعیت به paid
↓
مشاهده سفارش در پروفایل
↓
دیدن وضعیت سفارش و کد رهگیری
```

## Admin Flow واقعی

```text
ورود ادمین
↓
داشبورد فروش، سفارش‌ها، محصولات
↓
افزودن / ویرایش / حذف محصول
↓
ذخیره محصول روی سرور در data/products.json
↓
دیدن سفارش‌های ثبت‌شده در data/orders.json
↓
تغییر وضعیت سفارش: paid / processing / shipped / delivered / cancelled
↓
ثبت کد رهگیری ارسال
↓
نمایش کد رهگیری برای کاربر در پروفایل
```

## Profile Flow

ورود کاربر

## نکته production

این نسخه برای اجرای عملیاتی سبک روی سرور شخصی آماده‌تر شده است و داده‌های محصولات/سفارش‌ها را روی فایل‌های JSON در پوشه `data/` ذخیره می‌کند. برای فروشگاه پرترافیک یا چند ادمین همزمان، مرحله بعدی تبدیل همین فایل‌های JSON به SQLite/PostgreSQL است. منطق فلوها طوری نوشته شده که این انتقال ساده‌تر باشد.

## چک‌لیست قبل از لانچ

```text
1. مقدار JWT_SECRET را در .env production تنظیم کن.
2. اکانت‌های starter را در production فعال نکن مگر عمدی باشد.
3. عکس‌های local را با عکس‌های واقعی محصول جایگزین کن.
4. دامنه و SSL را روی سرور ایران تنظیم کن.
5. اگر پرداخت واقعی می‌خواهی، callback زرین‌پال را کامل به flow order وصل کن.
6. از پوشه data/ بکاپ روزانه بگیر.
```
